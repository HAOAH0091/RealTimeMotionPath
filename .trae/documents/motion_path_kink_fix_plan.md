# 修复计划：Motion Path 手柄折角问题

## 1. 问题分析
用户反馈在拖动 `AUTO` 类型的运动路径手柄时，如果进行大幅度位移，手柄会出现折角（左右不对齐）。
经过代码分析，问题可能出在 `move_handle_point` 函数中对手柄类型的处理逻辑不够健壮：
1.  当前代码仅在初始类型为 `AUTO` 时才强制转换为 `ALIGNED`。
2.  在连续拖动过程中（后续帧），类型已经是 `ALIGNED`，代码可能不再强制检查和修正类型。
3.  如果在此过程中，对侧手柄类型意外变为 `FREE`（或未正确设置为 `ALIGNED`），`update_opposite_handle` 函数就会跳过对侧更新，导致折角。
4.  此外，`update_opposite_handle` 内部的向量长度检查也可能在极端情况下导致更新失效。

## 2. 解决方案
为了彻底解决折角问题，我们将增强 `move_handle_point` 的逻辑，实施**强制对齐策略**：

### 核心改动
在 `move_handle_point` 函数中：
1.  **放宽类型检查**：不仅针对 `AUTO`，只要原始类型是 `ALIGNED` 或 `AUTO`，都视为“需要保持对齐”。
2.  **强制类型同步**：在每一帧拖动中，都强制将左右手柄类型设置为 `ALIGNED`，防止任何意外的类型漂移。
3.  **强制联动更新**：确保 `update_opposite_handle` 在对齐模式下必定执行，计算并修正对侧手柄位置。

### 预期效果
无论用户如何大幅度拖动手柄，只要初始状态是 `AUTO` 或 `ALIGNED`，左右手柄将始终保持在一条直线上（共线），彻底消除折角现象。

## 3. 执行步骤
1.  修改 `__init__.py` 文件中的 `move_handle_point` 函数。
2.  更新类型判断逻辑，包含 `ALIGNED` 类型。
3.  确保在设置手柄位置后，对侧手柄被正确更新。
