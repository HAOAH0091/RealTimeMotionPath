# Motion Path Pro 优化需求文档

## 1. 需求概述
本此优化旨在提升 Motion Path Pro 插件的视觉体验和功能实用性。主要关注点在于增强路径信息的表达（如速度可视化）、提供用户自定义样式的能力，以及优化现有的视觉元素（如手柄形状）。

## 2. 详细功能需求

### 2.1 路径帧点绘制 (Frame Points on Path)
**目标**：在绘制的运动路径上增加代表每一帧的点，通过点的疏密程度直观地展示动画速度（点越稀疏速度越快，点越密集速度越慢）。

**功能描述**：
*   在路径线上，对应每一帧的位置绘制一个小圆点。
*   **参考效果**：Blender 原生运动路径（Motion Paths）的样式。
*   **交互**：这些点仅作视觉参考，暂不需要交互功能（不需要被选中或拖拽，仅关键帧点可交互）。

**技术分析**：
*   目前插件已有 `position_cache` 缓存了每一帧的世界坐标。
*   需要在绘制路径线（Line Strip）的同时，遍历缓存数据绘制点（Points）。
*   建议使用 `GPU_POINT_SMOOTH` 或 `draw_billboard_circle` 绘制圆形点。

### 2.2 自定义样式设置 (Customizable Styles)
**目标**：允许用户根据个人喜好或特定背景需求，调整路径、手柄和点的视觉样式。

**设置项清单**：
用户可以在插件面板中调整以下参数：

1.  **路径线 (Path Line)**
    *   **粗细 (Width)**：调整路径连线的宽度。
    *   **颜色 (Color)**：调整路径连线的颜色（支持 RGBA）。

2.  **路径帧点 (Frame Points)**
    *   **显示/隐藏 (Show/Hide)**：开关，控制是否显示普通帧的点。
    *   **大小 (Size)**：调整普通帧点的大小。
    *   **颜色 (Color)**：调整普通帧点的颜色。

3.  **关键帧点 (Keyframe Points)**
    *   **大小 (Size)**：调整关键帧点的大小（通常比普通帧点大）。
    *   **颜色 (Color)**：调整关键帧点的颜色（区分普通状态和选中状态）。

4.  **手柄 (Handles)**
    *   **连线粗细 (Line Width)**：调整手柄连线的宽度。
    *   **连线颜色 (Line Color)**：调整手柄连线的颜色
    *   **端点大小 (Endpoint Size)**：调整手柄末端控制点的大小。
    *   **端点颜色 (Endpoint Color)**：调整手柄末端控制点的颜色（区分普通状态和选中状态）。

**UI 变更**：
*   在 `Motion Path Pro` 面板中新增一个“样式设置 (Style Settings)”折叠菜单或弹窗，容纳上述参数。

### 2.3 手柄端点形状优化 (Handle Endpoint Shape)
**目标**：统一视觉语言，将手柄端点的形状由正方形改为圆形，使其看起来更现代且与关键帧点风格一致。

**功能描述**：
*   将手柄末端的绘制逻辑从 `draw_billboard_square`（正方形）替换为 `draw_billboard_circle`（圆形）。
*   保持原有的选中高亮逻辑（选中时有发光/轮廓效果）。

## 3. 技术实现计划

### 3.1 数据结构更新
*   在 `MotionPathState` 或 `WindowManager` 中新增用于存储样式配置的 `Property Group`。

### 3.2 绘制逻辑更新 (`draw_motion_path_overlay`)
1.  **路径线绘制**：
    *   读取用户设置的线宽和颜色。
    *   应用 `gpu.state.line_width_set` 和 `shader.uniform_float`。

2.  **帧点绘制**：
    *   新增循环遍历 `_state.position_cache`。
    *   排除关键帧位置（避免重叠绘制），在其余帧位置调用 `draw_billboard_circle`。

3.  **手柄绘制 (`draw_motion_path_handles`)**：
    *   将 `draw_billboard_square` 替换为 `draw_billboard_circle`。
    *   读取用户设置的手柄大小和颜色。

### 3.3 性能注意事项
*   由于增加了帧点的绘制（可能是数百个点），需要确保使用 `batch_for_shader` 进行批量绘制，而不是逐个调用绘制函数，以维持视口流畅度。
*   建议将所有普通帧点合并为一个 Batch 进行一次性绘制。

## 4. 验收标准
1.  开启插件后，路径上显示每一帧的小圆点。
2.  用户可以通过面板修改路径颜色、粗细，且视口即时更新。
3.  用户可以修改手柄端点大小，且端点形状为圆形。
4.  修改样式配置后，性能无明显下降。
