# 优化计划：Motion Path Pro 骨骼模式交互修复

本计划旨在解决用户在测试骨骼模式运动路径时发现的两个问题：
1.  **选择切换问题**：在 Pose Mode 下切换选中骨骼时，运动路径未更新显示当前骨骼的路径。
2.  **模式切换问题**：从 Pose Mode 切换到 Object Mode 再切回后，运动路径消失。

## 问题分析

### 1. 选择切换问题
*   **原因**：`MOTIONPATH_AutoUpdateMotionPaths` 操作符的更新检测逻辑仅检查了“关键帧数据”的变化 (`_get_keyframe_values`) 和“关键帧选择”的变化 (`_has_selected_keyframes_changed`)。
*   **缺陷**：它**忽略了骨骼选择的变化**。在 Pose Mode 下，当用户切换选中的骨骼时，`active_object`（骨架对象）并未改变，且如果未修改关键帧，上述两个检查条件均不会触发更新。导致路径缓存 (`position_cache`) 仍保留旧骨骼的数据。

### 2. 模式切换问题
*   **原因**：`MOTIONPATH_DirectManipulation` 操作符包含一段逻辑，当检测到骨架处于 Object Mode 时，会强制关闭插件的核心开关：
    ```python
    wm.direct_manipulation_active = False
    wm.auto_sapty_active = False # 这会杀死自动更新操作符
    self.cancel(context) # 这会杀死交互操作符
    ```
*   **缺陷**：这种“一刀切”的清理逻辑导致用户只是临时切换模式查看（而非想关闭功能）时，插件被完全禁用。切回 Pose Mode 时，由于操作符已停止，路径无法自动恢复。

## 解决方案

### 1. 修复选择更新逻辑
修改 `MOTIONPATH_AutoUpdateMotionPaths` 类，增加对骨骼选择状态的监控。

*   **新增方法**：`_get_bone_selection_state(context)`
    *   获取当前 `active_pose_bone` 的名称。
    *   获取所有 `selected_pose_bones` 的名称列表。
    *   返回一个可哈希的元组（指纹）。
*   **修改 `modal` 方法**：
    *   在每一帧检查中，加入对 `_get_bone_selection_state` 的比对。
    *   如果骨骼选择发生变化，设置 `self._needs_update = True` 触发缓存重建。

### 2. 优化模式切换行为
修改 `MOTIONPATH_DirectManipulation` 类，使其在 Object Mode 下保持“休眠”而非“自杀”。

*   **修改 Object Mode 检测块**：
    *   **移除** `wm.direct_manipulation_active = False`
    *   **移除** `wm.auto_sapty_active = False`
    *   **移除** `self.cancel(context)`
    *   **改为**：仅 `return {'PASS_THROUGH'}`。
*   **效果**：当用户切到 Object Mode 时，交互操作符暂停响应但不退出，自动更新操作符继续运行（或保持待机）。当切回 Pose Mode 时，一切即刻恢复正常。

## 执行步骤

1.  **编辑 `__init__.py`**：
    *   定位 `MOTIONPATH_AutoUpdateMotionPaths` 类。
    *   实现骨骼选择状态检测逻辑。
    *   集成到 `modal` 循环中。
    *   定位 `MOTIONPATH_DirectManipulation` 类。
    *   修改 `modal` 中的 Object Mode 退出逻辑。

2.  **验证**：
    *   在 Pose Mode 下选中骨骼 A，显示路径；切换到骨骼 B，路径应立即更新。
    *   保持路径开启，切换到 Object Mode，再切回 Pose Mode，路径应保留且可交互。
