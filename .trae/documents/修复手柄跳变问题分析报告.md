# 手柄跳变问题深度分析报告

## 1. 核心问题总结
用户反馈的手柄“跳变”现象，实际上是由三个深层逻辑缺陷共同作用导致的。这些缺陷不仅涉及数学计算错误，还涉及对 Blender F-Curve 内部机制的误解和处理不当。

## 2. 根本原因一：数学计算错误 (Ratio Bug)
**影响范围：** 所有 ALIGNED 类型的手柄，尤其是当两侧关键帧时间间隔不一致时。

**现象：** 拖动一侧手柄，对侧手柄长度发生突变。

**分析：**
- 代码在 `update_opposite_handle` 函数中，强制将拖动侧的**数值增量 (Value Delta)** 直接赋给对侧。
- 忽略了**时间增量 (Time Delta)** 的影响。
- 正确的物理逻辑是保持**斜率 (Slope)** 一致，而非数值增量一致。
- **公式修正：** $\Delta Value_{Right} = \Delta Value_{Left} \times (\Delta Time_{Right} / \Delta Time_{Left})$

## 3. 根本原因二：手柄类型逻辑缺陷 (Handle Type Revert Bug)
**影响范围：** 所有 AUTO, AUTO_CLAMPED, VECTOR 类型的手柄。

**现象：** 拖动后手柄“回弹”或“瞬移”回原来的位置/方向，或者出现不可预测的抖动。

**分析：**
- 代码在拖动过程中，虽然临时将手柄类型修改为 `FREE` 或 `ALIGNED` 以允许修改。
- **致命错误：** 在函数结束前，代码试图将手柄类型**还原**为原始类型。
  ```python
  # 错误逻辑示例
  keyframe.handle_left_type = original_left_type  # 还原为 VECTOR/AUTO
  fcurve.update()  # Blender 重新计算 VECTOR/AUTO 手柄，覆盖用户修改
  ```
- **后果：** 用户的拖动操作实际上是为了打破原有的自动计算（例如打破矢量对齐，或自定义自动手柄的角度）。代码强行还原类型后，`fcurve.update()` 会立即触发 Blender 的内置计算逻辑，将手柄重置回自动计算的位置，导致用户的修改无效或产生“拉扯感”。

**修复方案：**
- 当用户手动修改了 AUTO/VECTOR 手柄时，应将其类型**永久更改**为 `ALIGNED` 或 `FREE`，而不能还原。这是 Blender 的标准行为逻辑。

## 4. 根本原因三：代码路径不一致 (Inconsistent Logic Paths)
**影响范围：** 取决于用户点击的具体位置（手柄端点 vs 手柄杆）。

**现象：** 不同的操作方式（点击圆点 vs 点击杆）导致不同的跳变行为。

**分析：**
- 代码存在两套完全独立的手柄移动逻辑：
  1. `move_handle_point`: 用于处理点击手柄端点（圆点）的情况。
  2. `move_selected_handles`: 用于处理点击手柄杆或其他判定区域的情况。
- **差异：**
  - `move_handle_point` 包含将 AUTO 转换为 ALIGNED 的逻辑（虽然实现有缺陷）。
  - `move_selected_handles` 根本**不包含**此转换逻辑。
- **风险：** 这种不一致意味着如果用户通过不同的方式触发拖动，会遇到完全不同的 Bug 表现（有的能拖动，有的拖不动，有的会回弹）。

## 5. 总结与修复计划 (Plan of Action)

### 5.1 统一代码逻辑 (Code Unification)
- 废弃 `move_selected_handles` 或 `move_handle_point` 中的一套逻辑，合并为一个统一的 `move_handle_unified` 函数。
- 确保所有触发点都调用这个统一函数。

### 5.2 修复数学计算 (Fix Math)
- 在统一函数中，彻底重写 `update_opposite_handle` 逻辑。
- 引入 `time_ratio = dt_right / dt_left`。
- 使用 `delta_value_opposite = delta_value_source * time_ratio`。

### 5.3 修正手柄类型行为 (Fix Handle Type Behavior)
- 删除所有“还原原始手柄类型”的代码。
- 确保当 AUTO/VECTOR 手柄被修改时，其类型**永久变更**为 `ALIGNED` 或 `FREE`。
- 如果用户按住特定修饰键（如 Shift），才考虑允许打破 ALIGNED 变为 FREE。

此修复方案不仅能解决当前的跳变问题，还能彻底消除潜在的逻辑隐患，提升操作的稳定性和一致性。
